# 咖啡书屋高保真 UI 重设计规范

## 设计目标

本次重设计保留现有前台与后台的全部业务功能、页面入口、数据字段、接口流程和权限逻辑，只重做视觉系统与响应式表达。当前版本以参考站 [gills-stomp-49159532.figma.site](https://gills-stomp-49159532.figma.site/) 为视觉基准：白色顶部导航、咖啡杯线性标识、咖啡馆大图 Hero、金色主按钮、半透明数据面板、米色内容背景、图文服务卡片与深色页脚。

## 字体系统

| 层级 | 字号 | 行高 | 权重 | 使用场景 |
| --- | --- | --- | --- | --- |

| H2 | `32px / 26px` | `1.12` | `850` | 页面标题、后台模块标题 |
| H3 | `18px` | `1.2` | `800` | 卡片标题、表格区块标题 |

## 组件规范

| 组件 | 样式 | 状态 |
| --- | --- | --- |
| Button Primary | 金色实底、8px 圆角、42px 高 | hover 提亮金色；disabled 56% opacity |
| Button Ghost | 半透明白底、深咖文字、轻边框 | hover 提高边框和背景浓度 |
| Button Danger | 浅红背景、红色文字 | hover 红色实底 |
| Card | 白色 92% 面板、1px 边框、8px 圆角 | hover 上移 2px 并增强阴影 |
| Form | 42px 高输入框、8px 圆角、蓝色 focus ring | invalid 使用浅红背景和红边 |
| Table | 分离边框、上屏横向滚动、表头 uppercase | 行 hover 使用浅灰蓝底 |
| Tabs / Segment | 8px 圆角轻按钮 | active 使用浅咖底和棕色文字 |
| Seat | 1:1 稳定尺寸 | free/success、occupied/danger、reserved/warning、selected/primary |
| Modal | 中央浮层、背景 blur、最大高度滚动 | 删除确认保留危险按钮 |
| State | 48px icon + 文案结构 | loading/error/empty 使用统一颜色语义 |

## 页面覆盖

### 前台

| 页面 | 保留功能 | 新视觉重点 |
| --- | --- | --- |
| 首页 | Hero、运营推荐、新闻信息流、核心入口 | 参考站同款咖啡馆大图首屏、白色居中文字、金色 CTA、半透明 KPI、图文服务卡 |
| 品牌介绍 | 品牌内容展示 | 与首页同源的高级内容页视觉 |
| 咖啡文化 | 咖啡商品展示、购物车入口 | 商品卡片统一为白底高密度信息卡 |
| 精品书库 / 详情 | 分类筛选、搜索、详情 | 书卡、标签、详情图文左右布局 |
| 文创商城 | 搜索、分类、数量、加入购物车、立即购买 | 工具条 + 稳定商品卡 + 库存/价格强调 |
| 购物车 / 结算 / 支付 / 支付成功 | 数量同步、库存校验、订单创建、支付审核状态 | 左工作区 + 右摘要卡，支付页用深色高对比卡 |
| 在线预约 / 我的预约 | 日期、时间、人数、座位图、取消 | 语义座位状态色，表单 sticky 到桌面右侧 |
| 活动 / 活动详情 | 容量、报名、提前报名 | 活动事实卡、报名 CTA 和容量信息清晰化 |
| 社区 / 发帖 / 详情 / 用户主页 | 发帖、图片、评论、点赞、隐私 | 信息流卡片、头像、审核状态统一 |
| 会员中心 / 资料 / 安全 / 积分 / 礼券 / 收藏等 | 会员数据、积分、隐私、礼券状态 | 数据卡、表单、礼券二维码弹层统一 |

### 后台

| 模块 | 保留功能 | 新视觉重点 |
| --- | --- | --- |
| 管理员登录 | admin 登录、token 存储 | 与前台统一，但更加工具化 |
| 工作台 | 指标卡、实时日志 | 数据卡高对比，表格密度提高 |
| 实时日志 | 搜索、筛选、分页、详情 | 左侧模块导航 + 顶部工具条 |
| 用户 / 商品 / 图书 / 订单 / 预约 / 活动 | 增删改、搜索、筛选、分页 | 表格、操作按钮、Modal 表单一致 |
| 订单审核 / 评论审核 | 通过、驳回、删除 | 语义状态 Badge 和危险操作按钮 |
| 内容 / 收入 / 数据库 / 数据看板 | 公告维护、统计、表结构 | 管理台信息密度和可扫描性优先 |

## 响应式规则

| 断点 | 布局规则 |
| --- | --- |
| Desktop `>1120px` | 参考站式顶部横向导航、全宽 Hero、三列服务卡；后台左侧固定导航 |
| Tablet `700-1120px` | 顶部导航换行，后台导航转横向滚动，主要双栏变单栏 |
| Mobile `<700px` | Logo 与账号动作同排，导航变紧凑横向入口，Hero 居中，服务卡单列，深色页脚垂直堆叠 |

## 数据同步说明

- 未修改 Vue 路由、Pinia store、API 封装、后台 `adminRequest`、表单字段名或 `data-testid`。
- 前台商品、购物车、订单、预约、社区、会员数据仍沿用原有接口和响应结构。
- 后台管理模块仍复用 `/api/admin/summary`、实时日志、CRUD、支付审核、评论审核等现有接口。
- 本次同步修改集中在 `front/src/styles.css`，属于视觉层同步，不改变数据读写路径。
