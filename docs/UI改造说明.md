# Coffee Book 全站 UI 改造说明

## 改造目标

本轮 UI 改造在不修改数据库结构、API 路径、API 返回结构、JWT 流程、Pinia 数据结构、Router 路由结构和后端业务逻辑的前提下，将“咖啡书屋”从基础页面升级为前后台统一的商业级展示系统。

改造重点：

- 前台形成咖啡书屋主题的会员、电商、预约、社区和内容体验。
- 后台形成运营管理系统风格，增强工作台、核心业务管理、高级运营看板和配置页。
- 统一 Design System、卡片、按钮、状态标签、Toast、Modal、Drawer、空状态与加载状态。
- 修复页面横向滚动风险，补充移动端断点适配。

## 设计系统

| 类型 | 说明 |
| --- | --- |
| 主色 | 咖啡棕与金色强调，延续咖啡书屋品牌感 |
| 背景 | 米白背景 + 白色卡片，降低页面压迫感 |
| 卡片 | 统一圆角、边框、柔和阴影和 hover 上浮 |
| 状态 | 成功、警告、危险、默认状态统一使用 Badge |
| 交互 | 表单 focus、按钮 hover/active/disabled、Toast、确认弹窗统一 |
| 响应式 | 桌面多栏，平板收缩，移动端单栏，禁止横向溢出 |

核心样式集中在 `front/src/styles/design-system.css`。

## Mock 与派生字段

缺少后端字段时，仅在前端展示层派生，不写入数据库：

- 订单时间轴、发货状态、预计积分。
- 收藏/笔记/消息分类、标签和消息已读状态。
- 积分任务、积分明细展示、礼品过期状态。
- 安全评分、登录设备、操作日志和风险提示。
- 后台内容趋势、收入来源占比、驾驶舱图表、角色权限、系统配置和存储状态。
