# 咖啡书屋平台项目说明

## 项目背景

咖啡书屋平台面向校园、社区或独立书店的复合经营场景。线下书屋不只需要展示图书和饮品，还需要处理座位预约、活动报名、文创商品销售、会员积分和社区互动，因此本项目把这些业务整合成一个可演示、可运行的全栈系统。

本项目适合作为课程设计或毕业设计展示，原因是它具备完整的前后端分离结构、真实业务流程、数据库持久化、权限校验、自动化测试和部署文档。

## 项目目标

- 完成一个前台用户端和后台管理端同时可用的咖啡书屋平台。
- 覆盖注册登录、商城下单、支付审核、座位预约、活动报名、社区互动、会员积分等核心业务。
- 使用 MySQL 保存核心数据，并支持首次运行自动建库建表。
- 通过接口测试和 E2E 测试验证关键流程，便于课程验收和答辩展示。

## 技术栈

- 前端位于 `front` 目录，使用 Vue 3 + Vite 构建用户端和后台管理端。
- 后端位于 `backend` 目录，使用 Node.js 原生 `http` 实现接口服务。
- 数据层使用 MySQL，后端启动时会自动创建数据库表并同步演示数据。
- 工程根目录提供统一脚本，支持一键启动、构建、lint、接口测试和 E2E 测试。

## 系统角色

- 游客：浏览首页、图书、商品、活动、社区内容，可以查看座位状态。
- 普通用户：登录后可使用购物车、创建订单、支付、预约座位、发帖评论、维护会员资料。
- 管理员：通过独立后台入口登录，可维护平台数据并审核订单、预约和社区内容。

## 核心功能

1. 用户登录或注册。
2. 浏览商品并加入购物车。
3. 创建订单并进入支付流程。
4. 后台审核订单支付状态。
5. 用户获得积分和会员成长值。
6. 用户可预约座位、参加活动、发布社区内容。
7. 管理员在后台维护运营数据并查看看板。

## 数据库说明

数据库名称默认为 `coffee_book`。后端会根据 `backend/.env` 中的配置连接 MySQL，并自动创建主要业务表：

- `users`：会员用户与个人资料。
- `admins`：后台管理员账号。
- `products`：咖啡饮品和文创商品。
- `books`：图书展示数据。
- `carts`：购物车记录。
- `orders`、`order_items`：订单与订单明细。
- `reservations`：座位预约。
- `activities`、`activity_applications`：活动与报名记录。
- `posts`、`comments`：社区帖子与评论。
- `notices`：后台公告内容。
- `audit_logs`：后台实时日志和操作记录。

## 项目亮点

- 前台和后台都可独立访问，适合展示完整系统而非单页 Demo。
- 业务模块丰富，覆盖商城、预约、社区、会员和后台运营。
- 后端接口做了登录态和资源归属校验，管理员接口与普通用户接口隔离。
- MySQL 持久化和 `.env` 配置让项目更接近真实部署形态。
- Playwright E2E 测试覆盖真实浏览器流程，并按运行批次清理临时测试数据。

## 答辩建议

答辩时可以按“业务背景 -> 系统架构 -> 核心流程 -> 数据库设计 -> 安全与测试 -> 展示页面”的顺序讲解。演示时建议先展示首页和商城，再完成一次下单或预约，最后切到后台管理端说明运营侧如何处理数据。
