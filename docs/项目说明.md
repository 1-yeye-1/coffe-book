# 项目说明

## 项目名称

咖啡书屋平台

## 项目定位

本项目是一个适合毕业设计、课程设计和答辩展示的全栈 Web 项目。系统以城市咖啡书屋为业务背景，将咖啡消费、图书阅读、文创商城、座位预约、活动赛事、书友社区和后台运营管理整合为一个完整平台。

## 使用对象

- 普通用户：浏览内容、购买文创商品、预约座位、参与活动、发布社区内容、管理会员资料。
- 后台管理员：维护商品、书籍、订单、支付、预约、活动、社区、公告、用户和系统数据。
- 答辩评审：通过前后台流程查看业务完整度、技术实现、界面完成度和测试验证结果。

## 核心业务流程

1. 用户注册或登录。
2. 浏览首页、咖啡文化、图书、商品、活动和社区内容。
3. 将商品加入购物车，提交订单并进入支付页。
4. 后台管理员审核支付记录，订单状态同步变化。
5. 用户预约座位或报名活动，后台可查看和管理。
6. 用户发布社区内容和评论，后台可进行审核。
7. 用户在会员中心查看积分、礼品、收藏、笔记、消息、个人资料和安全设置。

## 技术特点

- 前端使用 Vue 3 + Vite + Pinia + Vue Router。
- 后端使用 Node.js 原生 HTTP 服务，按前台接口和后台接口拆分路由。
- 使用 JWT 区分普通用户和管理员。
- 支持 MySQL 8 数据持久化，也支持测试环境内存数据。
- 提供后端 API 冒烟测试与 Playwright E2E 测试。
- 已完成全站 UI 重构和最终验收巡检。

## 最终验收结论

截至最终验收阶段，前台 16 个页面、后台 16 个模块均完成巡检。构建、接口测试和端到端测试全部通过，页面无 console error、无 console warning、无图片 404、无破图、无页面级横向滚动。项目已达到毕业设计展示标准。
