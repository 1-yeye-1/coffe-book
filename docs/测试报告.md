# 测试报告

## 测试目标

本项目的测试目标是验证咖啡书屋平台具备可交付性：代码无明显语法问题，前端可构建，后端核心接口可运行，关键浏览器业务流程可完整执行，并且自动化测试产生的临时数据不会污染开发数据。

## 测试命令

| 命令 | 覆盖范围 |
| --- | --- |
| `npm run lint` | package 配置、脚本语法、冲突标记、基础工程检查 |
| `npm run build` | Vue/Vite 前端生产构建 |
| `npm run test` | 后端接口冒烟测试 |
| `npm run test:e2e` | Playwright 真实浏览器端到端测试 |

## 后端接口测试覆盖

- 管理员登录。
- 后台数据看板读取。
- 未带管理员 token 访问后台接口会失败。
- 管理员商品、用户、订单、预约、活动、社区管理。
- 用户注册、密码登录、短信登录。
- 未登录访问会员接口会失败。
- 普通用户 token 访问后台接口会失败。
- 商品浏览、加入购物车、创建订单、提交支付。
- 订单支付状态只能由订单所属用户查询。
- 座位预约、我的预约、后台预约管理。
- 预约只能由所属用户取消。
- 社区发帖、评论、点赞、后台评论审核。
- 活动创建和活动报名。
- 测试结束后清理本轮创建的订单、预约、活动、帖子、商品和临时用户。

## E2E 测试覆盖

- 首页可以访问并展示首屏。
- 未登录访问会员中心会跳转到登录页。
- 登录表单非法输入会展示错误提示。
- 管理员登录并新增商品。
- 后台删除操作会出现确认弹窗，取消后数据保留。
- 用户登录、商品浏览、加入购物车、创建订单、跳转支付页面。
- 座位预约流程。
- 社区发帖流程。
- 后台订单管理搜索、分页控件。
- E2E 结束后按 `E2E_RUN_ID` 清理临时商品、订单、帖子和预约，并断言无残留。

## 测试数据隔离

E2E 测试创建的数据都会包含当前运行批次的 `E2E_RUN_ID`，例如：

- `E2E Coffee Kit <run-id>`
- `E2E Reading Note <run-id>`
- `E2E Reservation <run-id>`

清理逻辑只删除包含当前 `E2E_RUN_ID` 的数据，不会删除演示账号、正式商品或人工创建的数据。
