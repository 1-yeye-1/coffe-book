## 问题与修复

| 问题 | 修复 |
| --- | --- |
| 根目录一键启动在 Windows + Node 22 下调用 `npm.cmd` 可能 `spawn EINVAL` | 改为直接用 Node 启动前端脚本 |
| E2E 外层 cleanup 偶发 `fetch failed` | 将本轮数据清理放入 Playwright 测试内部，并断言清理结果 |
| 后端大文件中重复校验、会员和社区逻辑较多 | 抽出 `validators`、`audit`、`modules/member`、`modules/community`、`modules/admin-summary` |
| MySQL 常用查询缺少显式索引 | 增加启动时索引检查与创建 |
| 500 错误可能暴露底层错误信息 | 未捕获异常只在后端终端记录，前端返回友好提示 |
| 本机未启动 MySQL 时自动化测试无法启动后端 | 测试脚本默认注入 `COFFEE_BOOK_MEMORY_DB=1`，正常开发/生产启动仍连接 MySQL 自动建库 |
| E2E 支付提交断言匹配到多个元素 | 收窄 Playwright 选择器，仅断言支付处理提示或错误提示可见 |
