# 数据库设计

## 数据库名称

默认数据库名称为 `coffee_book`，可通过 `backend/.env` 中的 `DB_NAME` 修改。后端首次启动会自动创建数据库、业务表、索引和演示数据。

## 表结构说明

| 表名 | 用途 | 主要字段 |
| --- | --- | --- |
| `users` | 前台会员用户 | `id`、`name`、`phone`、`password`、`level`、`points`、`profile_public`、`favorites`、`notes`、`notifications`、`gifts` |
| `admins` | 后台管理员 | `id`、`name`、`account`、`password`、`role` |
| `products` | 咖啡饮品和文创商品 | `id`、`name`、`description`、`price`、`stock`、`category`、`image` |
| `books` | 图书展示 | `id`、`title`、`author`、`category`、`ranking`、`summary`、`publisher`、`published_at` |
| `carts` | 购物车记录 | `id`、`user_key`、`product_id`、`quantity`、`created_at` |
| `orders` | 订单主表 | `id`、`user_id`、`user_name`、`total`、`status`、`payment_method`、`payment_review_status`、`created_at` |
| `order_items` | 订单明细 | `id`、`order_id`、`product_id`、`name`、`price`、`quantity` |
| `reservations` | 座位预约 | `id`、`user_id`、`phone`、`seat_id`、`date`、`time`、`people`、`purpose`、`note`、`status` |
| `activities` | 活动赛事 | `id`、`title`、`capacity`、`applied`、`date`、`time`、`registration_start`、`early_start`、`location`、`description` |
| `activity_applications` | 活动报名记录 | `id`、`activity_id`、`user_id`、`phone`、`people`、`kind`、`created_at` |
| `posts` | 书友社区帖子 | `id`、`user_id`、`author`、`title`、`content`、`image`、`liked_by`、`likes` |
| `comments` | 社区评论 | `id`、`post_id`、`user_id`、`user`、`content`、`likes`、`liked_by`、`status` |
| `notices` | 内容公告 | `id`、`title`、`summary`、`date` |
| `audit_logs` | 实时日志 | `id`、`actor_type`、`actor_id`、`actor_name`、`action`、`target_type`、`target_id`、`detail`、`created_at` |

## 表之间的关系

- `users.id` 与 `orders.user_id`、`reservations.user_id`、`posts.user_id`、`comments.user_id`、`activity_applications.user_id` 关联。
- `orders.id` 与 `order_items.order_id` 关联。
- `products.id` 与 `order_items.product_id`、`carts.product_id` 关联。
- `activities.id` 与 `activity_applications.activity_id` 关联。
- `posts.id` 与 `comments.post_id` 关联。
- `admins.id` 与 `orders.payment_reviewed_by` 可关联后台审核人员。

## 索引设计

后端启动时会检查并创建常用查询索引：

- `orders.user_id`：查询当前用户订单。
- `orders.status`：后台按状态筛选订单。
- `reservations.user_id`：查询当前用户预约。
- `reservations.date`、`reservations.date + time`：查询座位时段状态。
- `posts.user_id`：查询用户主页动态。
- `comments.post_id`、`comments.user_id`：查询帖子评论和用户评论。
- `activity_applications.activity_id`、`activity_applications.user_id`：查询活动报名。
- `carts.user_key`：查询用户购物车。
- `audit_logs.created_at`：后台实时日志按时间排序。

`users.phone` 和 `admins.account` 使用唯一约束，天然具备索引能力。

## 初始化数据说明

项目保留 seed 演示数据，包括：

- 演示前台用户 `13800000000 / coffee123`。
- 演示后台管理员 `admin / admin123`。
- 商品、图书、预约、活动、社区帖子、公告和实时日志示例。

自动化测试创建的数据会带有测试批次标识，并在测试结束后通过管理员接口清理，不会删除 seed 演示数据。
