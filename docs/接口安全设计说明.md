# 接口安全设计说明

## 统一响应格式

后端所有业务接口统一返回：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

错误时 `success` 为 `false`，`data` 为 `null`。`backend/src/shared/response.js` 为 400、401、403、404、409、429、500 等常见状态码提供默认提示，便于前端统一展示错误。

## 鉴权设计

- 用户端接口使用用户 JWT，后台接口使用管理员 JWT。
- `/api/admin/*` 继续保持管理员 token 鉴权。
- 普通用户即使带用户 token，也不能访问后台接口。
- 下单、支付、订单详情、预约取消、社区操作等接口会校验用户归属。

## 参数校验

`backend/src/shared/validators.js` 提供通用校验函数：

| 函数 | 用途 |
| --- | --- |
| `validPhone` | 手机号格式 |
| `validEmail` | 邮箱格式 |
| `validStrongPassword` | 强密码 |
| `validInteger` | 整数范围 |
| `validDateString` | 日期格式和是否允许过去日期 |
| `validTimeLabel` | 时间或时段格式 |
| `validTextLength` | 文本长度 |
| `validEnum` | 枚举值 |

已补充校验的关键接口包括：

- 登录、注册、短信验证码发送。
- 创建订单、支付创建、支付提交、支付取消。
- 创建预约、取消预约。
- 活动报名。
- 社区发帖、评论。
- 后台订单、预约、活动、社区内容维护。

## 支付审核安全

- 用户只能为自己的订单创建支付记录。
- 用户提交“我已支付”后，订单进入 `payment_review`，不会直接变成 `paid`。
- 管理员在 `/admin/payments` 确认后，支付记录写入 `confirmed_at`，订单写入 `paid_at`。
- 管理员驳回后，支付记录变为 `failed`，订单回到待支付或取消状态。

## 测试数据隔离

后端冒烟测试和 Playwright E2E 均使用带时间戳的测试前缀创建临时商品、订单、预约和帖子。测试结束后通过后台删除接口清理，避免影响课程演示数据。
