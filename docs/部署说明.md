# 部署说明

## 环境要求

- Node.js 18 或更高版本。
- MySQL 8 或兼容版本。
- 生产环境建议准备 Redis，用于验证码和临时校验数据存储。

## 安装依赖

在项目根目录执行：

```bash
npm install
cd front && npm install
cd ../backend && npm install
```

## 配置后端环境变量

复制示例配置：

```bash
cd backend
copy .env.example .env
```

填写数据库和安全配置：

```text
PORT=4173
JWT_SECRET=replace-with-a-long-random-secret
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=dev
DB_PASSWORD=your-mysql-password
DB_NAME=coffee_book
REDIS_URL=redis://127.0.0.1:6379
```

注意：

- `JWT_SECRET` 生产环境至少 32 位。
- 不要把真实数据库密码提交到仓库。
- `NODE_ENV=production` 时必须配置 `REDIS_URL`。

## 本地开发启动

推荐在根目录一键启动：

```bash
npm run dev
```

脚本会同时启动：

- 后端：`http://localhost:4173/api`
- 前端：`http://localhost:5173`
- 后台：`http://localhost:5173/admin.html`

如果手动启动，建议先后端、再前端：

```bash
cd backend
npm run dev
```

另开终端：

```bash
cd front
npm run dev
```

## 生产构建

在根目录执行：

```bash
npm run build
```

构建产物位于 `front/dist`。后端可继续通过 `node backend/index.js` 启动，前端静态文件可部署到 Nginx、Apache 或其他静态服务。

## 反向代理建议

可将前端静态站点部署到 `https://your-domain.com`，后端 API 代理到 `/api`：

```text
/api -> http://127.0.0.1:4173/api
```

前端环境变量可配置：

```text
VITE_API_BASE=https://your-domain.com
```

## 验收命令

部署或提交前建议执行：

```bash
npm run lint
npm run build
npm run test
npm run test:e2e
```

所有命令通过后，再进行提交或上线。
