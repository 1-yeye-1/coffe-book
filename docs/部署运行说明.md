# 部署运行说明

## Node.js 版本要求

建议使用 Node.js 18 或更高版本。当前项目已在 Node.js 22 环境下验证根目录一键启动、构建和测试命令。

## MySQL 版本要求

建议使用 MySQL 8 或兼容版本。数据库字符集使用 `utf8mb4`，便于保存中文业务文案、评论、昵称等内容。

## 安装依赖

在项目根目录执行：

```bash
npm install
npm --prefix front install
```

如需单独安装后端依赖：

```bash
cd backend
npm install
```

## 配置数据库

复制环境变量示例：

```bash
cd backend
copy .env.example .env
```

按本机 MySQL 修改：

```text
PORT=4173
NODE_ENV=development
JWT_SECRET=replace-with-a-long-random-secret
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=dev
DB_PASSWORD=your-mysql-password
DB_NAME=coffee_book
REDIS_URL=redis://127.0.0.1:6379
```

本地开发没有 `.env` 时，后端会使用开发默认值。生产环境必须配置可靠的 `JWT_SECRET` 和数据库密码；如果 `NODE_ENV=production`，建议配置 Redis。

## 启动后端

在根目录执行：

```bash
npm run dev:backend
```

后端默认地址：

```text
http://localhost:4173/api
```

首次启动会自动创建 `coffee_book` 数据库、业务表、索引和演示数据。

## 启动前端

另开一个终端，在根目录执行：

```bash
npm run dev:front
```

前端默认地址：

```text
http://localhost:5173
```

后台管理入口：

```text
http://localhost:5173/admin.html
```

## 一键启动

也可以在根目录直接执行：

```bash
npm run dev
```

脚本会同时启动后端和前端。

## 访问地址

- 前台用户端：`http://localhost:5173`
- 后台管理端：`http://localhost:5173/admin.html`
- 后端 API：`http://localhost:4173/api`

## 常见问题

1. `npm run dev` 报端口占用：
   关闭占用 `4173` 或 `5173` 的进程，或修改环境变量 `PORT`、`FRONT_PORT`。

2. 后端提示 MySQL 连接失败：
   检查 MySQL 服务、账号密码和 `backend/.env` 配置。

3. 短信验证码看不到：
   开发环境验证码打印在后端终端，不会返回给浏览器。

4. 后台无法访问：
   请直接打开 `http://localhost:5173/admin.html`，并使用 `admin / admin123` 登录。

5. 生产环境 Redis 报错：
   配置 `REDIS_URL`，或确认 Redis 服务可连接。
